ENTITY COMPONENT ​

Actualizado a 17/2

Cómo esta estrcuturado ​

Hay 3 archivos esenciales, ec.h , component.h y entity.h, en todos ellos las clases desarrolladas están dentro del namespace ec. Por ello a la hora de utilizar las clases de este namespace se debe poner tal que así: ec:: NombreClase

Ec.h ​

Este archivo tiene la lista de componentes de todo el juego así como todos los grupos existentes. Al crear cualquier clase que herede de Component es necesario añadir en la lista de componentes el nombre del nuevo componente añadido. Ambas cosas están definidas de la siguiente manera:

cpp
#define _CMPS_LIST_ TRANSFORM, IMAGE, ENEMY_AI,PLAYER_MOVEMENT,PLAYER_INPUTS,NUM_COMPONENTS
#define _GRPS_LIST_ PLAYER,NUM_GROUP

SIEMPRE MANTENER QUE NUM_GROUP Y NUM_COMPONENTS SEAN LOS ÚLTIMOS ELEMENTOS

Component.h ​

Component ​

Este archivo tiene la definición de la clase base Component de la cual van a heredar todos los componentes que se creen. Esta clase tiene 2 métodos esenciales:

  • setContext(Entity* e) -> esta clase se encarga de establecer a que entidad pertence el componente, cada vez que se cree un componente se debe llamar a este método con la entidad a la que pertenece.
  • initComponent() -> este es un método virtual que se deberá usar si hay que inicializar atributos del componente que no se puede asegurar que se puedan inicializar en el constructor porque quizá dependan de que exista otro componente.

Render, Update y Collider Components ​

Estas clases son clases abstractas que tienen método render , update y checkCollision respectivamente. Si se quiere hacer un componente que tenga update deberá heredar de forma múltiple tanto de Component como de UpdateComponent, así con el resto de clases también.

Explicación detallada de ColliderComponents Ej:

class Image : public ec::Component,public ec::RenderComponent
Si heredas de Update, Render o Collider debes declarar obligatoriamente update(), render() o checkCollision() respectivamente

Entity.h ​

En esta clase se desarrolla como añadir, quitar y consultar los componentes de una entidad, así como los distintos atributos que todas las entidades tienes que sirven para el renderizado y actualización.

Metodos relacionados a componentes ​

A este método le pasas el componente a añadir a la entidad y los parámetros que este necesite:

cpp
//todo esto en el init de una escena (de momento);
auto e = addGameObject();
e->addComponent<Image>(&resource().images().at("fighter"));

para removeComponent, getComponent y hasComponent no es necesario pasar parámetros

cpp
//todo esto en el init de una escena (de momento);
auto e = addGameObject();
e->getComponent<Image>();//devuelve un puntero al componente 
e->removeComponent<Image>();
e->hasComponent<Image>();//comprueba si la entidad tiene dicho componente

Otros métodos ​

A la hora de crear una entidad se puede le debe pasar el grupoId al que pertenece, en caso de no pasarle ningúno este id será None. En caso de en la constructora no querer pasarle el grupo y querer que cambia de grupo en algún momento de la ejecución solo hay que llamar al método :

cpp
setGroup(ec::ent::groupID id) // el groupId debe ser uno de los establecidos en ec.h
