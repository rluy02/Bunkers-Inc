Metodos ​

Setters / Getters ​

Hay varios métodos clave:

  • Por una parte hay un setter para cada uno de los métodos de colisiones (onCollisionEnter,onCollisionExit,onTriggerEnter,onTriggerExit). Así se puede establecer desde fuera que hace cada uno de esos métodos.
  • setBodyPosition() -> método encargado de posicionar el collider según la entidad o de la posición dada, también setea el valor de _transformRelativePos así como la posición de _colliderDef.
  • setVelocity(Vector2D) -> método encargado de establecer la velocidad del body según el body que se le pasa
  • getLayer() y isTrigger() devuelven a que layer pertence el collider y si es trigger respectivamente

Construccion del collider ​

A la hora de construir un collider hay que hacerlo en el método initComponent de la clase que se haya creado (como BoxCollider por ejemplo). Pasos a seguir:

  • Establecer la definición del body cambiando los valores de los distintos atributos de _colliderDef y tras eso hacer:
cpp
_colliderId = b2CreateBody(_worldId, &_colliderDef);
  • Establecer la forma que queremos para el collider. Para hacer un Box o un circulo se haría de la siguiente forma:
cpp
// forma circular
b2Circle collider;
collider.center = b2Vec2_zero;
collider.radius = _radius / PIXELS_PER_METER; // se divide por los Pixeles por metro para que el tamaño se pase como lo que ocupa en pantalla

//forma cuadrada
b2Polygon collider = b2MakeBox(_size.getX()/PIXELS_PER_METER, _size.getY() / PIXELS_PER_METER);


// para crear otras formas mirad documentacion de la librería para ver todas las posibilidades de formas que hay (existe por ejemplo b2MakeOffsetBox para crear cajas con offset respecto a la posicion del body)
  • Establecer las propiedades del collider modificando los atributos de _colliderShapeDef. OBLIGATORIAMENTE AÑADIR LO SIGUIENTE:
cpp
//se establece si es o no trigger el collider
_colliderShapeDef.isSensor = _isTrigger;
//establece que pueda tener eventos de contacto
_colliderShapeDef.enableContactEvents = true;
//establece que pueda tener eventos de trigger
_colliderShapeDef.enableSensorEvents = true;
//se pasa como info del collider el propio objeto
_colliderShapeDef.userData = this;

//se le establecen las colisiones según la categoría
setCollisionFilter(_colliderShapeDef, _layer);

//Tras modificar el resto de propiedades que se quieran modificar hacer lo siguiente:
b2CreatePolygonShape(_colliderId, &_colliderShapeDef, &collider);
