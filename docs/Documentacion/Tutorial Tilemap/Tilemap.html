Crear un nuevo tilemap ​

El tilemap necesita una grid de habitaciones para poder renderizar elementos.

1. Crear el grid ​

Creamos un nuevo grid.

Generico:

Grid<type> grid = Grid<type>(fils, cols);

Ejempo para tilemap:

Grid<std::shared_ptr<Room>> grid = Grid<std::shared_ptr<Room>>(Tilemap::ROOM_GRID_SIZE, Tilemap::ROOM_GRID_SIZE);

2. Añadimos habitaciones ​

Ahora hay que añadir las habitaciones que va a tener el tilemap. Para añadir una habitación hay que llamar al constructor de room.

Generico:

grid.set(posX, posY, new Room(ROOM_ID, DOOR_MASK));

Ejemplo para tilemap:

grid.set(0, 0, std::make_shared<Room>(0, UP));

Door Mask ​

Cada habitación cuenta con una mascara binaria para saber donde hay puertas.

DirecciónBinario
UP0001
RIGHT0010
DOWN0100
LEFT1000
Se pueden combinar las puertas con el OR
  • Ejemplo: Puerta arriba y abajo -> 0001 | 0100 = 0101

3. Crear el tilemap y cargar ​

Ya terminada la grid se pasa al tilemap y se hace load.

Tilemap tilemap(grid);
tilemap.load(renderer);

Crear una variante de habitación en Tiled ​

Si quieres añadir un nuevo modelo de habitación es recomendable (no se si obligatorio) usar Tiled v1.10.2. Se puede descargar dese este link: https://github.com/mapeditor/tiled/releases/tag/v1.10.2

LO MÁS FACIL PARA CREAR UNA NUEVA HABITACIÓN ES DUPLICAR UNO DE LOS ARCHIVOS ROOM_X.JSON Y EMPEZAR DESDE AHÍ YA QUE EL TILESET TIENE PROPIEDADES ​

Requisitos ​

  • Las habitaciones tiene que ser de 10x10 tiles y cada tile de 32x32 píxeles (de momento)
  • Hay que usar el tilemap el tilemap que esta en: 2HN/assets/placeholders/dev_tileset.png
  • Hay que empotrar el tileset en el JSON.

Añadir puertas ​

Para añadir una puerta hay que colocar el tile en la capa walls que corresponde a la dirección de la flecha, es decir, puerta arriba tile de flecha arriba.

Si te fijas en la imagen, la capa Walls tiene propiedades personalizadas:

  • "wall_tile_id" - Indica el id del tile dentro del tileset (empezando por 1 esquina sup izq) para colocarlo como muro.
  • "floor_tile_id" - Lo mismo que el anterior pero para el suelo.

Estas propiedades se utilizan en el codigo para sustituir las puertas con muro(cerrado) o pared(abierto). Los nombres de las capas tiene que ser los mismos que el del tilemap_config.json.

Tilemap ​

El tilemap esta formado por dos Grid, una se encarga de almacenar la información de las habitaciones, Room. Y la otra es la grid absoluta, en la que se encuentran los tiles individuales del mapa.

  1. ROJO: es donde se colocan los tiles
  2. AMARILLO: cada abitación del mapa

Archivo de configuracion del tilemap ​

La configuración del tilemap se asigna desde un archivo json ubicado en:

  • 2HN/assets/config/tilemap/tilemap_config.json

Archivo de configuración:

{
  "grid": {
    "room_size": 10,  // Tamaño en tiles de la habitación
    "cell_size": 32   // Píxeles de los tiles
  },
  "layers": [         // Lista de capas de la habitación
       "Default",
       "Walls"
  ],
  "debug": {
    "tilemap_render_scale": 0.5,  // El factor de escalado del mapa (provisional)
    "render_debug_lines": false   // Si es true se dibuja la grid (imagen anterior)
  },
  "assets": {
    "rooms_path": "../assets/rooms/",  // Carpeta de los .json de las habitaciones
    "tileset_path": "../assets/placeholders/dev_tileset.png" // Tileset
  }
}

Grid ​

Hay una clase grid que es un template. Para el tilemap se usa para lo mencionado anteriormente.

Acceder a un valor de la grid: así se accede de forma segura

cpp
Value value = grid.at(x, y)

Asignar un valor a la grid: así se asigna un valor de forma segura

cpp
grid.set(x, y, value)

Room ​

La clase Room se encarga de guardar la información de la habitación, desde el número de puertas hasta el ID único de la habitación.

Para crear una nueva room hay que asignarle una posición, una ID y un número de puertas.
