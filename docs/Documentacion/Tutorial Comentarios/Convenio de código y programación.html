Convenio de código y programación ​

1. Idioma ​

  • Inglés : Para escribir código o nombrar cosas, funciones, clases, etc.
  • Español : Para escribir comentarios y aclaraciones.

2. Convenio de nombres ​

2.1 Miembros en una clase ​

“_” + camelCase

Ejemplo:

c
class Texture {
private:
    GameManager* _gameManager;
protected:
    Scene* _pauseScene;
public:
	int _numTexture;
}

2.2 Funciones y metodos ​

camelCase

Ejemplo:

c
class Scene {
private:
    vector<Entity>* getEntityList();
protected:
    bool checkStatus();
public:
	Scene* getScene();

2.3 Clase y Struct ​

PascalCase

Ejemplo:

c
class DynamicBitSet;
struct TypeList{};

2.4 Constantes ​

SCREAMING_SNAKE_CASE

Ejemplo:

c
constexpr int FRAME_PER_SECOND = 60;
const int WINDOW_WIDTH = 500;

2.5 Parámetros y variables temporales ​

camelCase

Ejemplo:

c
void checkStatus(std::vector<int> entityList, SDL_FRect hitBox){
    int numDisabled = 0;
}

3. Código de debugging ​

Es preferible no dejar código de debugging como por ejemplo std::cout ni cualquier ayuda de depuración suelto por el código. Si el código funciona como deberia y no se va a volver a tocar dicho código recordad borrar los códigos de debugging.

Si es necesario dejar el código de debugging por conveniencia, asegurad de que están de esta forma:

c
#ifdef _DEBUG
// Vuestro codigo de debugging
std::cout << "Something" << std::endl;
#endif

4. Namespace ​

Aunque es muy cómodo, no deberiamos using namespace std por ejemplo y para cualquier espacio de nombre, ya que puede provocar conflictos entre nombres sin querer.

Por otra parte, el buen uso de namespace nos puede ayudar con la aclaración de código.

5. Inicialización de variables ​

Siempre que podais inicializad los variables, ya sea una variable temporal, o la cadena de inicialización de una clase.
