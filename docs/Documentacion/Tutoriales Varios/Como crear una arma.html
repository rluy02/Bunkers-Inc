Como crear una arma

On this page

idea ​

La idea es tener una interfaz IWeapon, una clase template Weapon que hereda de la interfaz.

Esta clase tiene que recibir un comportamiento Behaviour como parametro de template,Behaviour su vez es un template tambien.

Behaviour recibe como parametetro de template un dato Data que necesita el comportamiento para que funcione. Esto es simplemente un struct que contiene informacion que necesita el comportamiento del ataque.

El Behaviour tiene que tener como obligatorio 3 metodos: initialize, instantiateAttack y update. A parte puede tener todos los metodos auxiliares o utiles que necesita.

Para entenderlo mejor este clase Weapon solamente actua como un contenerdor para un comportamiento de una arma, al contenedor no le importa que es lo que hace exactamente. Para generalizarlo, esto es algo que hace cierta cosa, lo que nos interesa es definir esa cierta cosa (el Behaviour) y no ese algo. El contenedor solo sabe que tiene los 3 metodos descritos arriba, y cuando le piede que haga algo simplemente tiene que llamar a esos metodos.

El Behaviour es una clase template, que recibe un Data. Esta clase puede ser independiente o heredado de otro Behaviour, siempre que sean compatible los datos que utilizan.

Como crear una arma ​

Un ejemplo de una arma a melee de fuego, que cuando ataca a un enemigo(intentar que sea independiente de la entidad quien ataca) tiene una probabilidad de quemar, la quemadura durara 5 segundos y hara cierto daño. Este arma atacara en forma de circulo, pero solo en un determinado angulo en el que esta mirando el atacante.

La idea es tener en cuenta que la forma ese collider puede ser variado.

Definir los datos ​

c
namespace combat
{
namespace data
{
    struct FireMeleeData // Aqui si existe un dato de melee basico, se puede heredarlo
    {
        using hitResolver =  FireMeleeCircleResolver; // esta explicado mas abajo.

        int damage;
        float attackSpeed;
        float burnChance;
        float burnDot;
        int fanAngle; // Angulo de apertura del ataque

        FireMelee() // Es recomendable tener un constructor por defecto
        {
            damage = 10;
            attackSpeed = 1.5;
            burnChance = 0.5;
            burnDot = 1;
            fanAngle = 90;
        }
    }  
}
}

Claro, hay que tener en cuenta que un melee puede atacar de varias maneras, mejor dicho, el collider del ataque puede ser variado, y para diferentes formas de collider quizas nos interesa diferenciar su forma de actuar, como puede ser el calculo del tiempo de impacto, detectar si esta realmente en la area de ataque, etc.

El punto: el mismo comportamiento de arma debe poder procesar cualquier tipo de variaciones que no afecta directamente al comportamiento en si. En este ejemplo, al comportamiento le tiene dar igual si el collider es un circulo, un rectangulo o una forma poligonal, tiene que poder procesarlo igualmente.

La idea es tener un objeto que se encarga de realizar el trabajo y el calculo de esas variaciones, y el comportamiento simplemente tiene que llamar al objeto para que realice los calculos y que devuelva los resultados que necesita. A esto lo llamo Resolver.

Este resolver simplemente se encarga de realizar los calculos, no debe saber nada mas que los datos que necesita, por lo tanto la idea es pasar los datos que el comportamiento desea calcular desde el propio comportamiento.

Definir el resolver (depende de casos concretos) ​

c
namespace combat
{
namespace resolver
{
    class FireMeleeCircleResolver 
    {
        public:
            ec::ColliderComponent createCollider(); // Devuelve el collider por ejemplo
            void triggerEnterCheck(); // un metodo para ver si esta realmente en la area de 
    }
}
}

Definir el comportamiento real ​

Como explicado arriba, el comportamiento es un template que recibe un dato Data

c
namespace combat
{
namespace behaviour
{
    template<typename Data>
    class FireMeleeBehaviour // Lo mismo, si existe una clase base por ejemplo Melee normal, se puede heredar sin problema
    {
        protected:
            Data _data; // guardamos el dato;

            ec::ColliderComponent* collider;

            typename Data::hitResolver _hitResolver; // explicado abajo

        public:

            /*
                constructor por defecto, se le puede pasar un FireMeleeData creado desde fuere, si no le pasamos nada utilizara el constructor por defecto de FireMeleeData.

                Como es un template parameter en realidad le puedes pasar cualqueier cosa.
            */
            FireMeleeBehaviour(const Data& data = Data());
            ~FireMeleeBehaviour();

            // Los 3 metodos obligatorios
            // La otra forma de implementarlo es tener la entidad, el AttackComponent como miembro del objeto, pero la decision ha sido que no, mejor 
            void initialize(entity_t entity, AttackComponent* controller, CollisionCategory layer);
            void update(entity_t entity, AttackComponent* controller, CollisionCategory layer);
            void instantiateAttack(entity_t entity, AttackComponent* controller, CollisionCategory layer);

        private:

            // se puede tener tantos metodos como necesita el comportamiento
            void calculateBurnChance();
    }
}
}
